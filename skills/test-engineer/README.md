# Test Generator

테스트 생성 작업을 위해 test-engineer 에이전트로 라우팅합니다.

## 사용법

```
/air-claudecode:test-engineer <대상 클래스 또는 메서드>
```

## 사용 시점

- 사용자가 테스트 작성, 테스트 생성, 테스트 커버리지 추가를 요청할 때
- 사용자가 "test", "테스트", "테스트 작성", "write tests", "test code"라고 말할 때
- 기존 프로덕션 코드에 대한 테스트 생성이 필요할 때

## 기능
- 의미 있는 테스트 생성 (비즈니스 로직, 엣지 케이스, 에러 경로)
- given-when-then 구조의 JUnit5
- AssertJ 플루언트 어설션, Kotest 매처
- 파라미터화 테스트, 중첩 테스트 구성
- Mockito-Kotlin / MockK 의존성 모킹

## 워크플로우

1. **이해** -- 프로덕션 코드를 읽고, 테스트 대상 식별
2. **계획** -- 테스트 범위 결정 (핵심 경로, 경계값, 에러 처리)
3. **테스트 작성** -- 프로젝트 컨벤션에 따라 의미 있는 테스트 작성
4. **실행 및 검증** -- 테스트 실행, 전체 통과 확인
